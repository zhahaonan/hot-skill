# hot-creator

**产品 x 热点内容策划助手** — AI Agent Skill for content creators.

采集全网热点趋势，结合你的产品/品牌/账号，生成完整的创作思路和素材（短视频脚本、小红书图文、长文大纲、素材清单、平台标题）。输出 Obsidian .md 笔记 + D3 交互式关系图谱。

## Quick Start / 快速开始

```bash
git clone https://github.com/zhahaonan/hot-skill.git
cd hot-creator
pip install -r requirements.txt
```

> 有 `uv` 的话更快：`uv pip install -r requirements.txt`
> **不需要配置 AI API Key** — Agent 自己就是 AI，自己做分析。

装好后对 Agent 说：

```
"我的产品是 XXX，帮我结合当下热点生成内容创作方案"
```

Agent 自动执行：采集全网热点 → 分析趋势 → 结合你的产品生成完整创作方案 → 输出 .md + 思维导图。

## 执行流程

```
Step 0: 获取画像 ──→ Step 1: 采集热点 ──→ Step 2: 分析趋势
                                                  │
                                                  ↓
Step 5: 告知结果 ←── Step 4: 导出文件 ←── Step 3: 生成方案
```

### Step 0 — 获取画像（必须先做）
Agent 从用户描述或 PDF 文档中提取结构化画像，包含：name, type, target_audience, usps, keywords, industry 等。

### Step 1 — 采集热点
采集 29 个平台的热门榜单 + 8 个实时新闻源，输出 `output/hotlist.json`。

### Step 2 — 分析趋势
Agent 自动去重、评分、分类，输出 `output/trends.json`。

### Step 2.5 — 获取全文（强烈建议）
用 WebFetch/WebSearch 获取热点原文，补充完整上下文。

### Step 3 — 生成内容方案
基于相关性评分，只对相关热点生成完整内容方案，输出 `output/briefs.json`。

### Step 4 — 导出文件
- `HotCreator/{date}/_Dashboard.md` — 每日概览
- `HotCreator/{date}/Topics/*.md` — 各话题详细笔记
- `output/mindmap.html` — 交互式思维导图

### Step 5 — 告知用户
总结 top 3 话题的创作方向，告知文件路径。

## 核心特性

### 1. 相关性评分框架

每个热点通过 5 维评分，总分 ≥ 6 分才生成内容：

| 维度 | 分值 | 评分标准 |
|------|------|----------|
| 用户人群重叠度 | 0-2 | 2=直接重叠 / 1=部分重叠 / 0=无重叠 |
| 情绪共鸣强度 | 0-2 | 2=强情绪触发 / 1=有情绪点 / 0=无情绪 |
| 产品功能承接度 | 0-2 | 2=直接解决痛点 / 1=间接相关 / 0=无关 |
| 平台传播适配度 | 0-2 | 2=天然适合 / 1=需要改编 / 0=不适合 |
| 风险等级 | 0-2 | 2=无风险 / 1=需谨慎 / 0=敏感话题 |

**评分结果**：
- ≥ 8 分 → 完整内容方案
- 6-7 分 → 完整方案 + 过渡逻辑
- < 6 分 → 不生成内容

### 2. Fallback 兜底模式

当热榜无合适切入点时，自动切换：

| 模式 | 说明 | 示例 |
|------|------|------|
| 行业趋势 | 从产品所在行业切入 | "2024年AI写作工具的5个趋势" |
| 用户情绪 | 从目标用户痛点切入 | "职场新人最焦虑的3件事" |
| Evergreen | 长青内容 | "如何用AI提高写作效率" |

### 3. 真实性保证

```json
{
  "data_points": [
    {"content": "XX功能上线3天，用户增长50%", "type": "fact", "source_url": "https://..."},
    {"content": "预计将继续增长", "type": "inference", "source_url": null}
  ],
  "sound_bites": [
    {"content": "这个功能绝了", "type": "suggestion"}
  ]
}
```

- `fact`: 有明确来源的事实（**必须带 source_url**）
- `inference`: 基于事实的合理推断
- `suggestion`: 建议/观点

**时效降权**：
- 超过 6 小时 → score -10
- 超过 12 小时 → score -20
- 超过 24 小时 → 不作为热点处理

### 4. 平台差异化模板

| 平台 | 开头 | 句长/风格 | CTA | 敏感表达 |
|------|------|-----------|-----|----------|
| 抖音 | 3秒 hook，反问/悬念/冲突 | 10-20 字，口语化 | 引导点赞/评论/关注 | 避免"第一""最""绝对" |
| 小红书 | 封面大字 + emoji + 数字 | 每段 2-3 句，闺蜜分享感 | 引导收藏/评论 | 避免医疗功效、过度营销 |
| 公众号 | 15-25 字，设悬念/反差 | 1500-3000 字，有观点 | 引导关注/在看 | 避免政治敏感、舆论操控 |
| 知乎 | 先抛观点，再用数据支撑 | 逻辑清晰，专业理性 | 引导赞同/评论 | 避免站队敏感话题 |
| B站 | 可玩梗，快速建立人设 | 有深度，年轻化 | 一键三连，弹幕互动 | 避免低俗、引战 |

### 5. 模型漂移自检

| Step | 自检项 | 不符合时的处理 |
|------|--------|---------------|
| Step 0 | 画像是否包含 name, type, target_audience, usps | 重新提取缺失字段 |
| Step 1 | 采集数量是否 ≥ 50 条 | 尝试更多平台或检查网络 |
| Step 2 | trends 数量是否 ≥ 10 条，每条是否有 summary, hot_window | 补充缺失字段 |
| Step 2.5 | Top 5 热点是否获取了全文或 context | 用 WebFetch/WebSearch 补充 |
| Step 3 | 每个方案是否通过相关性评分，所有字段是否填写 | 重新评分，补充空白字段 |
| Step 4 | 两个导出是否都成功 | 检查错误日志重试 |

### 6. 用户检查点

用户可以在以下检查点查看和修正问题：

| 检查点 | 时机 | 文件 | 快速修正命令 |
|--------|------|------|--------------|
| 画像确认 | Step 0 后 | Agent 内部状态 | "画像不对，我的产品是..." |
| 热点列表 | Step 2 后 | `output/trends.json` | "第 X 个话题不准确，应该是..." |
| 内容方案 | Step 3 后 | `output/briefs.json` | "话题 X 的产品结合点不对，应该从 XX 角度..." |
| 导出结果 | Step 4 后 | `HotCreator/{date}/` | "内容不够具体，补充话题 X 的脚本" |

## What You Get / 输出内容

每个热点话题生成**完整的、可直接执行的内容方案**：

| 内容 | 说明 | 示例 |
|------|------|------|
| 产品结合点 | 你的产品跟这个热点的真实连接 | "用户用ChatGPT时遇到的XX问题，正好我们的XX功能可以解决" |
| 创作角度 | 具体到能当标题，含完整执行步骤 | "ChatGPT新功能翻车？我用这个工具完美解决" |
| 短视频脚本 | 逐句话术 + 画面描述 + CTA | hook + 4-6 个节拍 + CTA |
| 小红书图文 | 封面标题 + 每页内容 + 标签 | 6-8 张图，每张含标题+正文+配图建议 |
| 长文大纲 | 章节 + 论据 + 产品植入点 | 4-5 个章节，每章含 heading/core_point/evidence |
| 素材清单 | 数据点、口播金句、封面文字、信源 | 每类至少 3 条，带来源标记 |
| 平台标题 | 抖音/小红书/公众号/知乎/B站各 2 个 | 各平台风格适配 |
| 发布建议 | 首发平台 + 最佳时间 + 窗口期 | "小红书首发，今晚 8 点，热度窗口 24 小时" |

**输出文件**：
- `HotCreator/{date}/_Dashboard.md` — 每日概览，top 话题一览
- `HotCreator/{date}/Topics/*.md` — 各话题详细笔记，拿来就能用
- `output/mindmap.html` — 交互式思维导图，可视化话题关联

## Harness 模式

此 Skill 采用 Agent Harness 架构：

1. **采集脚本** — 用 Task 子智能体执行，只返回文件路径，大数据不进主上下文
2. **Agent 分析** — Agent 自己做趋势分析和内容生成，参考 `reference/prompt-templates.md`
3. **相关性评分** — 自动评分过滤不相关热点，不强凑
4. **Fallback 兜底** — 无合适热点时自动切换模式，不卡死
5. **全文检索** — Step 2.5 用 WebFetch/WebSearch 获取热点原文，提升内容质量

## Self-Healing / 自修复

| 故障 | 自修复行为 |
|------|-----------|
| 单平台采集超时 | 内置 3 次 retry，指数退避 |
| 单个 export 失败 | 不影响其他 export，继续执行 |
| 依赖未安装 | 自动 pip install |
| 采集全失败 | 用 collect_rss 替代 |
| 无相关热点 | 自动切换 Fallback 模式 |

## 常见问题

### Q: 为什么有些热点没有生成内容？
A: 相关性评分 < 6 分的热点会被过滤，避免硬蹭热点。如果所有热点都不相关，会自动切换 Fallback 模式。

### Q: 如何修正不准确的画像？
A: 直接告诉 Agent："画像不对，我的产品是...，目标用户是...，核心卖点是..."

### Q: 导出的 MD 文件有些字段为空？
A: 这是模型漂移问题，Agent 在 Step 3 自检时会发现并补充。你也可以说："补充话题 X 的短视频脚本"。

### Q: 如何获取更好的内容质量？
A: Step 2.5 的全文检索很重要。确保网络畅通，Agent 会用 WebFetch/WebSearch 获取热点原文。
