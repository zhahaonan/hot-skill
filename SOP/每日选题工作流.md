# 每日选题工作流 SOP

## 概述

每天固定时间运行，完成从热点采集到内容策划的全流程。

## 触发方式

对 Agent 说："开始今日选题" 或 "帮我结合热点生成内容方案"

## 流程步骤

### Step 1：采集（约2-5分钟）

Agent 用 Task 子智能体执行：
- [ ] 热榜数据采集（微博/抖音/知乎/百度等 29 平台）
- [ ] RSS 订阅抓取（可选）
- [ ] 子智能体返回文件路径，大数据不进主上下文

**产出**：`output/hotlist.json`

### Step 2：趋势分析（约1-2分钟）

Agent 自己完成：
- [ ] 读取采集数据，跨平台去重
- [ ] 热度评分（0-100）
- [ ] 分类标注（科技/财经/娱乐/教育等）
- [ ] 趋势方向判断（上升/顶峰/下降/萌芽）

**产出**：`output/trends.json`

### Step 3：创作简报（约3-5分钟）

Agent 自己完成，结合产品信息：
- [ ] 判断产品与热点关联度
- [ ] 生成完整创作角度（具体到能当标题）
- [ ] 输出完整内容方案（短视频脚本/小红书图文/长文大纲）
- [ ] 输出完整素材清单（数据点/金句/标题/信源）

**产出**：`output/briefs.json`

### Step 4：导出（约1分钟）

Agent 用 Task 子智能体并行执行：
- [ ] Obsidian 笔记（Dashboard + 单篇话题笔记）
- [ ] Excel 报表
- [ ] 力导向关系图谱（HTML）

**产出**：
- `HotCreator/{date}/_Dashboard.md`
- `HotCreator/{date}/Topics/*.md`
- `output/report.xlsx`
- `output/mindmap.html`

## Step 5：人工筛选（5-10分钟）

- [ ] 打开 Dashboard，浏览今日话题总览
- [ ] 根据自身领域筛选 2-3 个值得做的话题
- [ ] 打开对应 Topic 笔记，选定创作角度
- [ ] 决定首发平台和发布时间

## Step 6：执行创作

- [ ] 根据选定角度和大纲开始创作
- [ ] 参考标题建议选择最终标题
- [ ] 按推荐时间发布

## 最佳实践

1. **固定时间**：建议每天 8:00-9:00 运行，在当天内容创作前完成
2. **不贪多**：每天聚焦 2-3 个话题，深度胜过广度
3. **关注萌芽**：emerging 话题竞争小，抢先发布优势大
4. **回看图谱**：每周看一次累积图谱，找到持续热点做系列内容
5. **复盘积累**：发布后记录数据，参见[[爆款复盘模板]]
