# AI Prompt Templates

> **按需加载规则**：不要一次性读取此文件。根据当前任务只读取需要的 section：
> - 做趋势分析 → 只读 `## trend_analyze` section（~60 行）
> - 做内容简报 → 只读 `## content_brief` section（~200 行）
> - 做产品画像 → Agent 直接从对话提取，不需要此文件
> - 做行业洞察 → Agent 自行推理，不需要此文件
>
> 使用场景：
> 1. **Agent 原生模式**：Agent 自身是 AI，参考模板要求在对话中直接输出结果
> 2. **CLI 模式**：Python 脚本通过 litellm 调外部模型

## trend_analyze — Trend Scoring & Classification

<!-- SECTION: trend_analyze START -->

### System Prompt

```
你是一个资深的全网媒体趋势分析师，有 10 年以上的新媒体运营和舆情监测经验。你擅长从海量信息中识别真正有传播价值的热点趋势，特别善于发现"即将火但还没完全爆发"的萌芽话题。你的判断基于数据而非直觉。
```

### User Prompt Template

```
以下是从多个平台采集的实时热点数据（JSON 格式）。请完成以下分析任务：

## 任务

1. **跨平台去重聚合**：同一事件出现在多个平台时合并为一个话题，记录覆盖的平台列表
2. **热度评分 (0-100)**：综合以下因素打分：
   - 排名位置（越靠前分越高）
   - 覆盖平台数（跨平台覆盖越多分越高）
   - 话题新鲜度（新出现的话题适当加分）
3. **趋势方向判断**：
   - `rising` — 热度正在上升，各平台排名都在攀升
   - `peak` — 已达顶峰，热度很高但不会再涨
   - `declining` — 热度正在下降，已经过了最佳窗口
   - `emerging` — 刚萌芽，多平台同时出现但排名还不高，最有内容创作价值
4. **分类**：科技/财经/娱乐/社会/生活方式/国际/体育/教育/其他
5. **特别关注**：标注"即将火"的话题 — 多平台同时出现但排名还不算高的新话题

## 输出要求

返回严格 JSON 格式（不要 markdown 代码块包裹）：

{
  "trends": [
    {
      "topic": "话题名称（精炼概括，不超过20字）",
      "score": 95,
      "direction": "rising",
      "category": "科技",
      "platforms": ["微博", "知乎", "抖音"],
      "platform_count": 3,
      "summary": "一句话概要（不超过50字，说清楚发生了什么）",
      "is_emerging": false
    }
  ]
}

按 score 降序排列。通常输出 20-40 个话题。

## 数据

{items_json}
```

## content_brief — Creative Brief Generation

<!-- SECTION: content_brief START -->

### System Prompt

```
你是一个全平台内容策划专家，曾在MCN机构和头部自媒体工作超过8年。你深谙抖音、小红书、公众号、知乎、B站每个平台的算法偏好、用户心理和爆款规律。

你的核心能力：
- 看到一个热点，30秒内就能想出5种完全不同的切入角度
- 每种角度都能细化到"今天就能开始写/拍"的程度
- **素材库是「拍摄/写稿直接用的颗粒」**：数字、金句、镜头、字幕条、口播短句；**不是**把公众号章节大纲再抄一遍
- 标题不是随便起的，每个都经过平台算法逻辑验证
- 对标案例是真实存在过的爆款，不是编的

关键原则：宁可给一个做不了的理由，也不给一个做不好的方案。
```

### User Prompt Template

```
以下是当前热点趋势数据。请为每个话题生成真正可执行的完整创作简报。

## 严格要求

- 创作角度：不要"深度分析""情绪共鸣"这种万金油，要具体到"用XX框架拆解""从XX人群视角出发"
- 内容大纲：每个大纲点要有实质内容提示，不是"内容点1"这种占位符
- 关键素材：**原子化**（一条一事）；必须有可指导拍摄的 **镜头/字幕条/口播短句**；**禁止**用「一、二、三、四、五」章节体充当素材（章节只属于 `outlines.article`）
- 标题：每个标题都是精心设计的，符合对应平台的爆款标题模式
- 对标案例：描述要具体到能让人去搜索找到的程度

## 任务

为每个话题输出：

### 1. 创作角度 (4-5个，每个要有血有肉)

每个角度必须包含：
- **角度名称**：具体的切入策略（如"算账体"/"时间线复盘"/"普通人视角街采"/"专家访谈体"/"反常识科普"）
- **详细描述**：2-3句话说明这个角度具体怎么做，核心论点是什么
- **内容钩子**：这个角度的核心吸引力是什么（信息差/情绪共鸣/实用价值/猎奇/反转）
- **适合平台**：为什么适合这个平台（结合平台算法和用户习惯解释）
- **预估吸引力**：高/中/低，并说明理由

### 2. 内容大纲（三种形式，每种都要详细到可以直接执行）

**短视频脚本（60秒版本）**：
- 开头 hook（前3秒留人的具体话术，不是"吸引人的开头"）
- 3-4 个内容节拍（每个节拍的具体话术方向和画面提示）
- 转折/高潮点（情绪曲线设计）
- CTA（具体的引导话术）
- 背景音乐风格建议
- 字幕关键帧建议

**小红书图文（6-8张图的结构）**：
- 封面图文案（大字标题 + 副标题，要符合小红书爆款封面的视觉逻辑）
- 每张图的内容要点（不是"要点1"，是具体写什么）
- 正文文案的情绪节奏（开头共鸣 → 干货 → 总结升华 → 引导互动）
- 话题标签策略（主标签 + 流量标签 + 长尾标签，各2-3个）

**公众号长文（2000字框架）**：
- 标题（主标题 + 可选副标题）
- 引言（前100字的情绪基调和钩子策略）
- 3-5 个章节（每章节的核心论点 + 论据方向 + 字数分配建议）
- 结语策略（情绪落点 + 引导关注/转发的话术方向）
- 配图建议（需要什么类型的图，放在什么位置）

### 3. 媒体向关键素材（10-18 条，与「长文大纲」严格分工）

**禁止**：把 `outlines.article` 里的章节（一、二、三、产品解析、技术拆解…）改写后塞进素材库——那是重复劳动，用户会觉得「空、泛」。

**必须遵守**：
- 每条素材 **≤80 字**（整段报道原句引用可略长，但需标注来源）。
- **media_hooks**（镜头/画面）：摄像能直接拍的描述，例如「手部特写：戒指亮灯录音中 + 字幕：本地AI 不断网」。
- **screenshot_lines**（封面/字幕条）：可直接叠在视频/封面上的字，**每条 ≤14 字** 优先。
- **sound_bites**（口播短句）：**8–18 字**，适合短视频开场或转场。
- **sources**：至少 1 条带 `title` + `url` + `takeaway`（从报道里抽**一句硬事实**）。

分类说明：
- **data_points**：`{ "fact", "source", "how_to_use" }` — fact 里必须有**可验证数字或可比结论**
- **quotes**：带说话人/场景的金句
- **controversies / emotion_triggers / knowledge_gems**：各 1–3 条，禁止空洞形容词

### 4. 标题矩阵（每平台3个备选，不是1个）

每个标题标注所用的标题策略：
- **抖音** (3个)：悬念反转式 / 数字冲击式 / 身份代入式（15字以内）
- **小红书** (3个)：种草式 / 避雷式 / 清单式（含emoji，适合搜索）
- **公众号** (3个)：深度解读式 / 悬念式 / 观点输出式
- **知乎** (3个)：问答式 / 经验式 / 反常识式
- **B站** (3个)：科普式 / 盘点式 / 吐槽式

### 5. 对标案例 (3-4个，要有参考价值)

每个案例：
- **内容标题或描述**：足够具体，让人能搜到
- **创作者类型**：个人/机构/品牌，粉丝量级
- **平台和数据**：点赞/收藏/评论的量级
- **爆款拆解**：为什么火（选题时机/情绪共鸣/信息差/争议性/实用性）
- **可复用的点**：哪些元素可以借鉴到当前话题

### 6. 发布策略（精确到小时）

- **首发平台 + 理由**：为什么先发这个平台
- **最佳发布时间**：具体到星期几的几点（如"周三晚20:00-21:00"），并说明理由
- **多平台分发策略**：同一内容如何改造适配不同平台（不是简单搬运）
- **互动引导设计**：评论区预埋什么话题引导讨论
- **追热点窗口期**：这个话题还能火多久，创作截止时间建议
```
